## Plan: Phase 0 Project Bootstrap and Configuration

### 1. Goal
Set up a bootable Rails 7.2 monolith in the current repository root with required gems, framework components, initializers, database setup, Devise and RSpec installation, and baseline verification commands passing.

### 2. Files affected
- [CREATE] Rails application scaffold files in the repository root (generated by `rails new .`).
- [MODIFY] `Gemfile`
- [MODIFY] `config/database.yml`
- [MODIFY] `config/application.rb`
- [CREATE] `config/initializers/openai.rb`
- [CREATE] `config/initializers/sidekiq.rb`
- [CREATE] `config/initializers/rack_attack.rb`
- [CREATE] `config/initializers/pagy.rb`
- [CREATE] `db/migrate/*_enable_pgvector.rb`
- [MODIFY] `spec/rails_helper.rb`
- [MODIFY] `config/credentials.yml.enc` (structure only, values remain placeholders)

### 3. Spec files to write first
None for Phase 0. This phase is framework bootstrap and configuration.

### 4. Database changes
- Enable PostgreSQL pgvector extension through migration `EnablePgvector`.
- Install Rails framework migrations generated by Action Text and Active Storage.

### 5. External side effects
- `bundle install` to install gems.
- `rails` generators for Action Text, Active Storage, Turbo, Stimulus, Devise, and RSpec.
- Database creation and migration commands.

### 6. Risks and open questions
- Running `rails new .` in a non-empty git repo may overwrite generated defaults. We will preserve `AGENTS.md`, `PROMPT.md`, and existing plan files.
- `rails credentials:edit` is interactive; credentials skeleton may need to be set manually if non-interactive editing is constrained.

### 7. Execution order
1. Generate Rails app in current repo root with Rails 7.2, PostgreSQL, Tailwind, Importmap.
2. Replace Gemfile with required dependency set and run `bundle install`.
3. Run installs/generators: Action Text, Active Storage, Turbo, Stimulus, Devise, RSpec.
4. Configure `config/database.yml`, `config/application.rb`, and required initializers.
5. Create and run pgvector enable migration, then run full `rails db:migrate`.
6. Update `spec/rails_helper.rb` with SimpleCov + WebMock setup.
7. Run Phase 0 verification commands (`rails server` boot check, `rspec`, `rubocop`, migration status).
